**HEALTHMETRO®**

**WhatsApp Automation Message Flow**

**Blood Collection Customer Registration System**

**Developer Documentation**

![]()

**1. PURPOSE**

This document defines the WhatsApp automation workflow and message templates for the Healthmetro® Blood Collection Customer Registration System.

The objective is to automatically send WhatsApp notifications after successful customer registration and appointment booking through:

* QR Code Registration
* Provider-specific Registration Link
* Unique client_id onboarding flow

![]()

**2. AUTOMATION TRIGGER**

**Event Name**

CUSTOMER_REGISTRATION_SUCCESS

**Trigger Conditions**

Execute WhatsApp notifications only after:

* Valid client_id verification
* Successful customer registration
* Successful appointment slot booking
* Unique Customer ID generation
* Database transaction completion

If any validation fails:

* Do NOT trigger WhatsApp notifications

![]()

**3. WHATSAPP NOTIFICATIONS REQUIRED**

The system must send WhatsApp notifications to:

| **Recipient**          | **Purpose**                              |
| ---------------------------- | ---------------------------------------------- |
| Healthcare Provider / Client | Inform provider about new registration         |
| Customer                     | Appointment confirmation and thank you message |
| Internal Operations Team     | Operational alert for coordination             |

![]()

**4. PROVIDER / CLIENT NOTIFICATION**

**Purpose**

Notify the healthcare provider that a customer has registered using their dedicated Healthmetro® QR code or registration link.

![]()

**Recipient Source**

Fetch from:

providers.whatsapp_number

![]()

**WhatsApp Template Name**

healthmetro_provider_registration_alert

![]()

**Message Template**

🟢 Healthmetro®

Dear {{provider_name}},

A new customer has successfully registered for blood sample collection through your Healthmetro® QR registration link.

Customer Details:

👤 {{customer_name}}

🆔 {{customer_id}}

📅 {{appointment_date}}

⏰ {{time_slot}}

🩸 {{collection_type}}

✅ Appointment confirmed successfully.

Thank you for your continued support and partnership with Healthmetro®.

![]()

**Dynamic Variables**

| **Variable** | **Database Source** |
| ------------------ | ------------------------- |
| provider_name      | providers.name            |
| customer_name      | customers.name            |
| customer_id        | customers.customer_id     |
| appointment_date   | appointments.slot_date    |
| time_slot          | appointments.slot_time    |
| collection_type    | customers.collection_type |

**12. FAILURE HANDLING**

**Retry Logic**

| **Attempt** | **Retry Delay** |
| ----------------- | --------------------- |
| Retry 1           | 1 minute              |
| Retry 2           | 5 minutes             |
| Retry 3           | 15 minutes            |

After maximum retries:

* Mark message as FAILED
* Notify admin dashboard
