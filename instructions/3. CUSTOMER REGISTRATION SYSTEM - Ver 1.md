__BLOOD COLLECTION CUSTOMER REGISTRATION SYSTEM__

__Unique Client ID Based Onboarding \(Revised\)__

__1\. SYSTEM OVERVIEW__

This system is used for __blood sample collection booking only__\.

Customers register under a healthcare provider using:

- QR Code OR
- Provider\-specific link containing client\_id

Provider selection is fully automatic\.

__2\. CORE RULE__

- Provider cannot be selected manually
- Provider is determined only via client\_id
- If client\_id is missing or invalid → block access
- All bookings are strictly tied to Client ID

__3\. ENTRY URL STRUCTURE__

https://yourdomain\.com/register?client\_id=CLIENT\-STATE\-YEAR\-XXXX

OR via QR code printed at provider location\.

- No manual provider selection

__15\. BUSINESS RULES__

- Provider mapping is automatic and fixed
- QR is the only entry source
- Collection type determines workflow
- One slot = one booking only
