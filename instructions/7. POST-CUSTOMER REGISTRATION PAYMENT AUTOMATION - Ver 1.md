__7\. PAYMENT STATUS HANDLING__

__CASE 1: SUCCESSFUL PAYMENT__

Webhook triggers:

payment\_status = SUCCESS

System actions:

- Activate booking
- Confirm appointment slot
- Notify provider/agent
- Trigger home collection routing \(if applicable\)

__CASE 2: PAYMENT FAILED__

System actions:

- Keep booking in HOLD state
- Send retry link
- Release slot after timeout \(configurable\)

__CASE 3: PAYMENT TIMEOUT__

If no payment within X minutes:

- Slot automatically released
- Booking marked EXPIRED
- Notification sent to customer
