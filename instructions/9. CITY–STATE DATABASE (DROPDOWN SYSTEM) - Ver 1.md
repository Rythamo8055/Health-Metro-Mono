__4\. API DESIGN \(DYNAMIC DROPDOWN\)__

__GET ALL STATES__

GET /api/states

Response:

\[

  \{ "state\_id": 1, "state\_name": "Tamil Nadu", "state\_code": "TN" \},

  \{ "state\_id": 2, "state\_name": "Karnataka", "state\_code": "KA" \}

\]

__GET CITIES BY STATE__

GET /api/cities?state\_id=1

Response:

\[

  \{ "city\_id": 101, "city\_name": "Chennai" \},

  \{ "city\_id": 102, "city\_name": "Coimbatore" \}

\]

__5\. FRONTEND DROPDOWN LOGIC__

__STEP 1: STATE DROPDOWN__

User selects:

- State

__STEP 2: AUTO LOAD CITIES__

On state selection:

fetch\('/api/cities?state\_id=selected\_state'\)

Populate city dropdown dynamically\.

__6\. UI BEHAVIOR__

__State Field:__

- Dropdown \(searchable\)
- Required

__City Field:__

- Disabled until state selected
- Auto\-populated

__7\. VALIDATION RULES__

- City must belong to selected state
- No free text allowed
- Only active states/cities visible
- Prevent mismatched combinations
